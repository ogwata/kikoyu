# デモ手順 (文字起こし＋話者分離のライブデモ)

kikoyu の話者分離を人前で見せるための手順です。**実運用 (`whisperx-work` 等) には触れず**、
公開リポジトリを別フォルダに clone して回します。

## デモ用サンプル音声の条件 (重要)

話者分離のデモなので、**1人の声では成立しません** (`SPEAKER_00` しか出ない)。
次を満たす「自分で録った多人数サンプル」を使ってください。

- **2人以上** (3人いると分離が視覚的に分かりやすい)。交互に話し、ときどき声が重なるとなお良い
- **2〜5分程度** (文字起こしは RTF ≒ 1.0。60分音声で約60分かかるため、ライブでは短尺に)
- **録音相手の同意があり、内容は当たり障りのないもの** (人前で流す = 公開になるため)
- 日本語だと `language="ja"` 前提とよく噛み合う

**使ってはいけないもの:**
- 本物の会議・町内会録音 (住民の声・実名 = 個人情報)
- ライセンス不明の市販コンテンツ (映画・TV・音楽・配信クリップ等)
- 固有名詞辞書の `prompts/initial_prompt.local.txt` (実名を含む。デモでは作らず、同梱の
  `initial_prompt.example.txt` のままにする)

## 準備 (本番前に一度やっておく)

1. 実運用と分けるため、別フォルダに clone する:
   ```bash
   cd ~
   git clone https://github.com/ogwata/kikoyu.git
   cd kikoyu
   cp .env.example .env        # HF トークンを書く → chmod 600 .env
   ```
2. **モデルキャッシュを事前に温める。** 初回は whisper-large-v3 + pyannote (合計数 GB) を
   ダウンロードするので、ライブ前に一度サンプルを通して完走させておく
   (ダウンロード待ちを観客に見せないため)。

## 当日の流れ

1. デモ用サンプルを clone したフォルダ直下に置く。
2. `kikoyu.ipynb` 冒頭の `AUDIO_FILE` にそのファイル名を書く
   (フォルダに1件しか音声が無ければ空のままでも自動検出される)。
3. `docs/setup.md` のとおりコンテナを起動し、Jupyter でセルを **Cell 1 (PyTorch パッチ) から順に**実行。
4. 完走すると `out/{stem}_transcript_speaker.txt` ができる。これを開いて見せる:
   ```
   [00:00:03] SPEAKER_00: ...
   [00:00:09] SPEAKER_01: ...
   [00:00:15] SPEAKER_00: ...
   ```
   `[HH:MM:SS] SPEAKER_XX:` で「誰がいつ何を言ったか」が分かれているのが一目で伝わる。
   話者数を事前に把握しておけば「3人入れたらラベルが3つに分かれた」と説明しやすい。

## 後片付け・プライバシー

- サンプル音声と `out/` の出力は `.gitignore` 済み。**コミットしない**。
- デモ後にサンプルを消すなら clone フォルダごと削除してよい (実運用とは別物)。

## コードだけ見せる場合

GitHub 上で README とノートブックを説明するだけなら、**Spark は不要**で、画面に出しても配っても安全
(リポジトリに個人情報は無い)。実際に動かして見せたいときだけ、上の手順で Spark の別フォルダ clone が要る。
